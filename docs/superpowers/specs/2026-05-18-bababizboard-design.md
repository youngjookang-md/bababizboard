# BabaBizBoard — 카카오 비즈보드 배너 제작 툴 설계 문서

## 개요

Streamlit 기반의 카카오 비즈보드(1029×258px) 배너 광고 소재 자동 제작 및 편집 툴.
상품 이미지 누끼 처리, 카피 입력, 로고 배치, 템플릿 선택, 프로젝트 저장/불러오기, PNG 다운로드를 지원한다.

---

## 카카오 비즈보드 공식 가이드 고정값

| 항목 | 값 |
|------|----|
| 캔버스 크기 | 1029 × 258 px |
| 파일 형식 | PNG-24 / PNG-32 |
| 용량 제한 | 300KB 이하 |
| 메인 카피 폰트 | Spoqa Han Sans Bold |
| 메인 카피 크기 | 48pt |
| 메인 카피 색상 | #4C4C4C |
| 서브 카피 폰트 | Spoqa Han Sans Regular |
| 서브 카피 크기 | 39pt |
| 서브 카피 색상 | #777777 |

> 폰트 크기·색상은 가이드 고정값이므로 사용자 변경 불가. 위치(X/Y)만 조정 가능.

## 모듈 상세 설계

### canvas.py — PIL 레이어 합성 엔진

레이어를 아래에서 위 순서로 합성한다:

```
1. 배경 (흰색 1029×258)
2. 상품 이미지 (누끼 PNG, X/Y/Scale 적용)
3. 로고 (누끼 PNG, X/Y/Scale 적용)
4. 메인 카피 텍스트 (Spoqa Han Sans Bold 48pt #4C4C4C)
5. 서브 카피 텍스트 (Spoqa Han Sans Regular 39pt #777777)
```

- 슬라이더 값이 변경될 때마다 전체 파이프라인 재실행
- 결과 이미지를 `st.image()`로 표시
- 다운로드용 PNG 바이트 생성 시 300KB 초과 여부 체크

### bg_remover.py — 배경 제거

- `rembg.remove(input_image)` 호출
- 처음 실행 시 모델 자동 다운로드 (~170MB, 1회)
- 처리 완료된 이미지는 세션 상태에 캐싱 (재업로드 시에만 재처리)
- 로고에는 "배경 제거" 체크박스 옵션 제공

### templates.py — 레이아웃 템플릿

3개 기본 템플릿을 딕셔너리로 정의:

| 템플릿명 | 설명 | 상품 이미지 위치 | 카피 위치 |
|---------|------|----------------|----------|
| `left_image` | 좌측 이미지형 | x=20, y=20 | x=450, y=80 |
| `right_image` | 우측 이미지형 | x=700, y=20 | x=50, y=80 |
| `text_focus` | 텍스트 강조형 | x=800, y=10 | x=50, y=60 |

셀렉트박스에서 템플릿 선택 시 → 각 레이어의 기본 X/Y/Scale 자동 세팅

### storage.py — 프로젝트 저장/불러오기

저장 형식 (JSON):

```json
{
  "name": "캠페인_A",
  "template": "left_image",
  "main_copy": "지금 바로 시작하세요",
  "sub_copy": "한정 특가 ~50% 할인",
  "product_image_path": "projects/캠페인_A/product.png",
  "logo_path": "projects/캠페인_A/logo.png",
  "layers": {
    "product": { "x": 20, "y": 20, "scale": 80 },
    "logo":    { "x": 850, "y": 10, "scale": 25 },
    "main_text": { "x": 450, "y": 80 },
    "sub_text":  { "x": 450, "y": 140 }
  }
}
```

- 저장: 프로젝트명 폴더 생성 → JSON + 이미지 파일 복사
- 불러오기: `projects/` 하위 폴더 목록을 셀렉트박스로 표시 → 선택 시 세션 상태 복원

### layer_controls.py — 슬라이더 UI

각 레이어별 조정 가능 항목:

| 레이어 | X | Y | Scale |
|--------|---|---|-------|
| 상품 이미지 | 0~900 | 0~200 | 10~200% |
| 로고 | 0~900 | 0~200 | 10~100% |
| 메인 카피 | 0~900 | 0~200 | — |
| 서브 카피 | 0~900 | 0~200 | — |

## 데이터 흐름

```
사용자 입력 (업로드/슬라이더/텍스트)
    ↓
st.session_state 업데이트
    ↓
canvas.py render() 호출
    ↓
PIL Image 생성
    ↓
st.image() 미리보기 표시 + 다운로드 버튼 바이트 제공
```

## 제약 및 예외 처리

- rembg 최초 실행: 스피너 + "모델 로딩 중..." 메시지 표시
- 300KB 초과: 경고 메시지 표시, 다운로드는 허용
- 이미지 미업로드 상태: 해당 레이어 렌더링 스킵
- 프로젝트명 중복 저장: 덮어쓰기 확인 다이얼로그

## 구현 우선순위

1. 캔버스 합성 엔진 + 기본 렌더링
2. 텍스트 입력 + 위치 슬라이더
3. 이미지 업로드 + 배치
4. 배경 제거 (rembg)
5. 템플릿 선택
6. 저장/불러오기
7. PNG 다운로드 + 용량 체크
